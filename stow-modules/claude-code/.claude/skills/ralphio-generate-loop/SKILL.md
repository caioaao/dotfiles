# Generate loop.sh

## What This Skill Does

Generates `loop.sh` (~50-80 lines) — an executable bash script that orchestrates the Ralph loop. It feeds prompt files to the CLI tool in headless mode, pushes after each iteration, and restarts with a clean context. It supports two modes: `build` (default) and `plan`.

loop.sh is NOT a process manager, monitoring system, or CI pipeline. It is a simple loop with a clean exit.

## Before Generating

Study the target project thoroughly before writing anything:

- CLI tool in use (claude or other)
- Existing automation scripts

- Prompt file names and locations
- Model preferences (opus, sonnet, etc.)

## What the Generated Script Must Accomplish

### Mode Selection
```bash
./loop.sh           # build mode (default)
./loop.sh build     # explicit build mode
./loop.sh plan      # planning mode
./loop.sh build 5   # build mode, max 5 iterations
```

### Core Loop Behavior
Each iteration:
- Feed the appropriate prompt file to the CLI tool via stdin (headless, non-interactive)
- Use appropriate CLI flags for autonomous operation (permissions, output format, model)
- Capture stdout to detect the `ALL_TASKS_COMPLETE` signal from the build prompt
- Git push after each iteration (the build prompt commits, the loop script pushes)
- Increment iteration counter when a max is specified

### Completion Detection
The build prompt prints `ALL_TASKS_COMPLETE` to stdout when the plan is empty or all tasks are done. The loop script must:
- Capture CLI output (tee to file or variable)
- Grep for the exact string `ALL_TASKS_COMPLETE`
- Stop the loop and report completion when detected
- In plan mode, always run exactly one iteration (planning is a single pass)

### Startup Checks
Before entering the loop:
- Verify the prompt file exists
- Verify the CLI tool is available
- Display a status banner: mode, prompt file, current branch, max iterations (if set)

### Signal Handling
Trap Ctrl+C for clean exit without corrupting state.

### Script Quality (Non-Negotiable)
- `#!/usr/bin/env bash` shebang
- `set -euo pipefail` for strict error handling
- Clear variable names and comments explaining intent
- Under 80 lines for the basic version

## What the Generated Script Must Exclude

- **Process management** — no daemonization, no PID files, no backgrounding
- **Log aggregation** — stdout/stderr is sufficient; don't build a logging system
- **Branch management** — user creates branches; the script just pushes current
- **Environment setup** — no dependency installation, no version checks beyond CLI availability
- **Cost tracking** — no token counting, no API cost monitoring
- **Retry logic** — if an iteration fails, stop and let the human investigate

## Parameterization

Discover and embed these project-specific values:
- CLI tool name and invocation flags (e.g., `claude --print --dangerously-skip-permissions`)
- Prompt file paths (`PROMPT_plan.md`, `PROMPT_build.md`)
- Model selection (may differ between plan and build modes)
- Git push strategy (simple push, or push with upstream tracking)

## Language That Activates Good Behavior

| Use this | Not this | Why |
|---|---|---|
| `set -euo pipefail` | Selective error handling | Fail fast, fail loud |
| `ALL_TASKS_COMPLETE` | Magic exit codes | Greppable, explicit, not lost in CLI exit code normalization |
| "Status banner" | "Startup message" | Implies structured, at-a-glance information |
| `trap cleanup EXIT` | Ad-hoc signal handling | Consistent cleanup on any exit path |

## Structure Template

```bash
#!/usr/bin/env bash
set -euo pipefail

# Configuration
CLI="[discovered CLI tool]"
BUILD_PROMPT="PROMPT_build.md"
PLAN_PROMPT="PROMPT_plan.md"
MODEL_BUILD="[discovered model preference]"
MODEL_PLAN="[discovered model preference]"
COMPLETION_SIGNAL="ALL_TASKS_COMPLETE"

# Parse arguments
MODE="${1:-build}"
MAX_ITERATIONS="${2:-0}"  # 0 = unlimited

# Select prompt and model
case "$MODE" in
  build) PROMPT="$BUILD_PROMPT"; MODEL="$MODEL_BUILD" ;;
  plan)  PROMPT="$PLAN_PROMPT";  MODEL="$MODEL_PLAN"; MAX_ITERATIONS=1 ;;
  *)     echo "Usage: $0 [build|plan] [max-iterations]"; exit 1 ;;
esac

# Startup checks
[[ -f "$PROMPT" ]] || { echo "Error: $PROMPT not found"; exit 1; }
command -v "$CLI" &>/dev/null || { echo "Error: $CLI not found"; exit 1; }

# Status banner
BRANCH=$(git branch --show-current 2>/dev/null || echo "unknown")
echo "=== Loop: $MODE | Prompt: $PROMPT | Branch: $BRANCH | Max: ${MAX_ITERATIONS:-unlimited} ==="

# Cleanup on exit
cleanup() { echo "Loop stopped."; }
trap cleanup EXIT

# Main loop
ITERATION=0
while true; do
  ITERATION=$((ITERATION + 1))
  echo "--- Iteration $ITERATION ---"

  OUTPUT=$("$CLI" [flags] --model "$MODEL" --print < "$PROMPT" 2>&1) || true
  echo "$OUTPUT"

  # Check for completion signal (build mode only)
  if [[ "$MODE" == "build" ]] && echo "$OUTPUT" | grep -q "$COMPLETION_SIGNAL"; then
    echo "All tasks complete."
    break
  fi

  # Push changes
  git push || echo "Warning: push failed"

  # Check iteration limit
  if [[ "$MAX_ITERATIONS" -gt 0 && "$ITERATION" -ge "$MAX_ITERATIONS" ]]; then
    echo "Reached $MAX_ITERATIONS iterations."
    break
  fi
done
```

## Anti-Patterns

- **Over-engineering error recovery**: Complex retry logic, exponential backoff, error categorization. If an iteration fails, stop. The human should investigate before burning more tokens.

- **Daemonization**: Running the loop in the background with PID files. This is a foreground script the human monitors in a terminal.

- **Cost tracking**: Counting tokens or dollars per iteration. This belongs in the CLI tool or a separate monitoring layer, not in a 60-line bash script.

- **Branch management**: Automatically creating feature branches, managing PRs, or rebasing. The user sets up their branch; the script just pushes.

- **Log file rotation**: Writing structured logs, rotating files, or parsing previous runs. Pipe to a file if you want logs: `./loop.sh build 2>&1 | tee loop.log`.

## Design Principles (ralphio system)

- **Generate-then-own**: This skill generates a tailored artifact. The user owns it from that moment.

- **Self-contained output**: loop.sh is a complete, runnable script with no external dependencies beyond the CLI tool.

- **Study before generating**: Discover the project's CLI preferences before producing output.

- **Simplicity over features**: The loop's power is in its simplicity and statelessness. Complexity in the orchestrator defeats the purpose — the intelligence is in the prompts, not the script.
