Commit the current changes.

Rules:
- Split the work into logical commits (by vertical slice or concern). Optimize for ease of review.
- Match this repo's existing commit-message style — check `git log --oneline -20` first.
- Do not add "Generated by" or co-author trailers unless the repo already uses them.
- Run `git status` before and after; show the final `git log --oneline` of the new commits.

Modifiers (parsed from `$@`):
- If `branch` appears → first create a new branch off the current upstream base before committing.
- If `push` appears → push after committing (set upstream if missing).
- If `pr` appears → after pushing, open a PR with `gh pr create`, deriving title/body from the commits.
- Anything else in `$@` → treat as additional instructions and honor them.

Arguments: $@
